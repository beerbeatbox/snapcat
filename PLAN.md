# Snapcat — Implementation Plan (Source of Truth)

Snapcat is a native macOS menu-bar screenshot app (Swift + SwiftUI + AppKit).
Flow: global hotkey ⇧⌘F4 → system region-selection UI → capture auto-copied to
clipboard → floating preview panel at bottom-left → "Edit" opens an editor
window with annotation tools (blur, numbered points, ellipse, rectangle, color
picker) → Copy / Save from the editor.

Design bar: CleanShot X-level polish — system materials, rounded corners, soft
shadows, SF Symbols, subtle animations. No third-party dependencies.

## Constraints

- Swift language mode **5.0** (avoid strict-concurrency friction; no `@MainActor` gymnastics needed — all UI code already runs on main).
- Deployment target **macOS 14.0**.
- **No** sandbox, **no** hardened runtime, ad-hoc code signing (`CODE_SIGN_IDENTITY = "-"`).
- No external packages. AppKit + SwiftUI + CoreImage + Carbon only.
- Do NOT use deprecated `NSImage.lockFocus` — use `NSImage(size:flipped:drawingHandler:)` for the final render.
- Do NOT `git init`, commit, or create a repo.
- Menu-bar-only app: `LSUIElement = true` (no Dock icon).

## Build system

Project is generated with **XcodeGen** (already installed). Create
`project.yml` at repo root, then verify with:

```bash
cd ~/Projects/Snapcat
xcodegen generate
xcodebuild -project Snapcat.xcodeproj -scheme Snapcat -configuration Debug build
```

**Acceptance: `xcodebuild … build` must exit 0.** Iterate until it does.
Do not launch the app (it needs Screen Recording TCC approval — the user will run it).

## Component specs

### AppDelegate.swift — entry point + menu bar

`@main final class AppDelegate: NSObject, NSApplicationDelegate` with a
`static func main()` that creates `NSApplication.shared`, assigns the delegate,
and calls `app.run()`. (No SwiftUI `App` scene — windows are managed manually.)

In `applicationDidFinishLaunching`:
- Create `NSStatusItem` (square length). Button image: SF Symbol `"cat.fill"`,
  fall back to `"camera.viewfinder"` if nil.
- Menu:
  - **Capture Area** — action calls `CaptureService.shared.captureArea()`.
    Show the ⇧⌘F4 shortcut as display-only key equivalent:
    `item.keyEquivalent = String(UnicodeScalar(NSF4FunctionKey)!)`,
    `item.keyEquivalentModifierMask = [.command, .shift]`.
  - **Edit Last Capture** — opens editor with `CaptureService.shared.lastCapture`;
    implement `NSMenuItemValidation.validateMenuItem` so it's disabled when nil.
  - separator, **About Snapcat** (`orderFrontStandardAboutPanel`), separator,
    **Quit Snapcat** (⌘Q).
- Wire hotkey: `HotkeyManager.shared.onHotkey = { CaptureService.shared.captureArea() }`
  then `HotkeyManager.shared.register()`.

### HotkeyManager.swift — global hotkey via Carbon

`import Carbon.HIToolbox`. Singleton. `register()`:
- `InstallEventHandler(GetApplicationEventTarget(), callback, 1, &eventType, userData, &handlerRef)`
  where `eventType = EventTypeSpec(eventClass: OSType(kEventClassKeyboard), eventKind: UInt32(kEventHotKeyPressed))`.
- The C callback cannot capture context — pass
  `Unmanaged.passUnretained(self).toOpaque()` as userData and recover with
  `Unmanaged<HotkeyManager>.fromOpaque(userData!).takeUnretainedValue()`,
  then call `onHotkey?()`. Return `noErr`.
- `RegisterEventHotKey(UInt32(kVK_F4), UInt32(cmdKey | shiftKey), hotKeyID, GetApplicationEventTarget(), 0, &hotKeyRef)`
  with `hotKeyID = EventHotKeyID(signature: 0x534E_4354 /* "SNCT" */, id: 1)`.
- Note: `cmdKey`/`shiftKey` are Carbon's Int constants — this is correct; do not
  use `NSEvent.ModifierFlags` here.

### CaptureService.swift — capture + clipboard

Singleton. `private(set) var lastCapture: NSImage?`.

`captureArea()`:
- Temp file: `FileManager.default.temporaryDirectory.appendingPathComponent("snapcat-<uuid>.png")`.
- `Process` running `/usr/sbin/screencapture` with args `["-i", path]`
  (interactive region selection — gives us the native crosshair UI and handles
  the Screen Recording TCC prompt; Snapcat is the responsible process).
- In `terminationHandler`, hop to main queue. If the file doesn't exist the user
  pressed Esc — do nothing. Otherwise load `NSImage(contentsOf:)`, store as
  `lastCapture`, copy to clipboard, delete the temp file, then
  `PreviewPanelController.shared.show(image:)`.

`static func copyToClipboard(_ image: NSImage)`:
`NSPasteboard.general.clearContents()` then `writeObjects([image])`.

### PreviewPanel.swift — floating thumbnail, bottom-left

`PreviewPanelController` (singleton, plain class) + `PreviewCard` SwiftUI view
in the same file.

Panel: `NSPanel(contentRect:styleMask:[.borderless, .nonactivatingPanel], …)`,
`isFloatingPanel = true`, `level = .statusBar`, `backgroundColor = .clear`,
`isOpaque = false`, `hasShadow = false` (the SwiftUI card draws its own shadow —
an NSWindow shadow around a clear window causes artifacts),
`collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]`.
Content: `NSHostingView(rootView: PreviewCard(...))`, size from `fittingSize`.
Position: bottom-left of `NSScreen.main!.visibleFrame` + 16pt margins.
Show with `orderFrontRegardless()` and a fade-in (`alphaValue` 0 → 1 via
`animator()`).

Behavior:
- Auto-dismiss after **6 s** (Timer). While the mouse hovers the card, pause the
  timer; on exit restart it at 3 s. Dismiss = fade out, then `orderOut`.
- `PreviewCard`: thumbnail `Image(nsImage:)` scaledToFit, max 300×200,
  `RoundedRectangle(cornerRadius: 12)` clip, 1pt white 25 %-opacity border
  overlay, `.shadow(color: .black.opacity(0.35), radius: 16, y: 6)`, outer
  `.padding(14)` so the shadow isn't clipped by the panel bounds.
- On hover, overlay appears (animated, 0.15 s): centered **Edit** button
  (`.borderedProminent`) and a small ✕ close button at top-leading in a
  `.ultraThinMaterial` circle.
- Edit → dismiss panel, then `EditorWindowController.open(image:)`.
- A new capture while a panel is visible replaces it (call dismiss first in `show`).

### Editor/EditorModels.swift

```swift
enum EditorTool: String, CaseIterable, Identifiable {
    case blur, number, ellipse, rectangle
    // id, SF symbol ("eye.slash", "1.circle", "circle", "rectangle"),
    // English label ("Blur", "Number", "Oval", "Box"),
    // KeyEquivalent shortcut ("b", "n", "o", "r")
}

struct Annotation: Identifiable {
    enum Kind {
        case blur(CGRect)
        case number(center: CGPoint, value: Int)
        case ellipse(CGRect)
        case rectangle(CGRect)
    }
    let id = UUID()
    var kind: Kind
    var color: Color   // SwiftUI Color; ignored for .blur
}
```

**All geometry is stored in image PIXEL coordinates with top-left origin.**
The view scales at render time; export draws 1 unit = 1 pixel. This is the
single most important invariant in the editor — keep it consistent everywhere.

### Editor/EditorViewModel.swift — `ObservableObject`

Init (failable) from `NSImage`:
- Extract `cgImage` via `image.cgImage(forProposedRect:context:hints:)` — this
  gives true pixel dimensions (retina screenshots have size-in-points ≠ pixels;
  we normalize by working in pixels everywhere).
- `pixelSize = CGSize(width: cg.width, height: cg.height)`.
- Precompute `blurredCG`: whole-image pixellation via CoreImage —
  `CIFilter.pixellate()` (import `CoreImage.CIFilterBuiltins`), input
  `CIImage(cgImage:).clampedToExtent()`, `scale = Float(max(10, pixelSize.width / 90))`,
  output `.cropped(to:)` original extent, rendered back with `CIContext`.
  Committed blur regions are just this image masked to the blur rects — cheap
  and identical in preview and export.
- Keep `baseNSImage = NSImage(cgImage: cgImage, size: pixelSize)` and
  `blurredNSImage` likewise (1 pt = 1 px) for export drawing.

Published state: `tool` (default `.rectangle`), `color` (default `.red`),
`annotations: [Annotation]`, `draft: Annotation?` (in-progress drag),
`justCopied: Bool` (transient "Copied ✓" feedback, auto-clears after ~1.5 s).

Derived: `nextNumber` = count of `.number` annotations + 1. Sizing helpers used
by BOTH preview and export so they match exactly:
- `lineWidthImage = max(4, pixelSize.width * 0.004)`
- `numberDiameter = max(32, pixelSize.width * 0.028)`

Interaction (called from the view with view-space points + current scale;
convert to image space by dividing by scale, clamped to image bounds):
- `dragChanged(startView:currentView:scale:)` — build/update `draft` per tool.
  For `.number` the draft follows the cursor (`center = current`,
  `value = nextNumber`). For rect-based tools, rect = normalized rect between
  the two points.
- `dragEnded(...)` — commit: `.number` always; rect-based only if width AND
  height > 3 px (in image space). Clear `draft`.
  (A plain click with the number tool must place a point — use
  `DragGesture(minimumDistance: 0)` so this works.)
- `undo()` — `annotations.popLast()`. Removing the latest number keeps the
  sequence consistent because `nextNumber` is derived from count.
- `blurRects(scale:) -> [CGRect]` — committed blur rects (+ draft blur rect, so
  the pixellation previews live during the drag), scaled to view space.

Export — `renderFinal() -> NSImage` using
`NSImage(size: pixelSize, flipped: true) { _ in … }` (flipped ⇒ top-left origin,
matching stored coordinates; AppKit `NSBezierPath` and `NSAttributedString.draw`
handle flipped contexts correctly):
1. `baseNSImage.draw(in: full rect)`.
2. Each blur rect `r`: draw `blurredNSImage` `in: r` with
   `from:` rect **vertically flipped** (`y = pixelH − r.maxY` — the source image's
   own space is bottom-left origin), `operation: .sourceOver`, `fraction: 1`,
   `respectFlipped: true`, hints `[.interpolation: NSImageInterpolation.none.rawValue]`.
3. Rect/ellipse: `NSBezierPath` stroke, `lineWidth = lineWidthImage`,
   color `NSColor(annotation.color)`.
4. Number: filled circle Ø `numberDiameter` in annotation color + white ring
   (~6 % of Ø), then the value as bold white text
   (`NSFont.boldSystemFont(ofSize: Ø * 0.55)`) centered via measured
   `NSAttributedString.size()`.

Actions:
- `copyResult()` — `CaptureService.copyToClipboard(renderFinal())`, set `justCopied`.
- `saveResult()` — `NSSavePanel`, `allowedContentTypes = [.png]`
  (import `UniformTypeIdentifiers`), default name
  `"Snapcat yyyy-MM-dd 'at' HH.mm.ss.png"`. PNG bytes via
  `tiffRepresentation → NSBitmapImageRep → representation(using: .png)`.

### Editor/EditorView.swift — SwiftUI UI

`EditorView(model:)` = VStack: toolbar / `Divider()` / canvas.
`.frame(minWidth: 560, minHeight: 400)`.

Toolbar (`.padding(H12 V9)`, background `.bar`):
- Tool group: segmented-look cluster — plain-style buttons 34×26 with SF symbol,
  selected one gets `Color.accentColor` rounded-6 background + white symbol;
  cluster sits on a `Color.primary.opacity(0.06)` rounded-8 backdrop. Each has
  `.help("<Label> (<key>)")` and `.keyboardShortcut(key, modifiers: [])`.
- `Divider().frame(height: 20)`.
- Color swatches: 8 preset circles (red, orange, yellow, green, blue, purple,
  white, black; ~16 pt, selected shows an accent ring) + compact
  `ColorPicker` (labels hidden). Dim + disable this group when tool == .blur.
- `Spacer()`, then: "Copied ✓" secondary text when `justCopied`;
  Undo button (`arrow.uturn.backward`, ⌘Z, disabled when no annotations);
  **Save** (`square.and.arrow.down`, ⌘S); **Copy** (`doc.on.doc`, ⌘C,
  `.borderedProminent`).

Canvas: `GeometryReader`; `scale = min(availW/pixelW, availH/pixelH, 1/screenBackingScale)`
(the backing-scale cap shows retina pixels at natural size instead of 200 %);
32 pt padding around the fitted image; backdrop
`Color(nsColor: .underPageBackgroundColor)`; the image gets a soft shadow.

Layered `ZStack(alignment: .topLeading)` at fitted size:
1. `Image(model.cgImage, scale: 1, label:)` resizable, `.interpolation(.high)`.
2. Blurred image (same sizing, `.interpolation(.none)`) masked by a `Shape`
   whose path is all `blurRects(scale:)` as rounded rects (radius 6).
3. `Canvas` overlay (`allowsHitTesting(false)`) drawing committed + draft
   annotations scaled to view space: strokes at `lineWidthImage * scale`;
   draft blur additionally gets a dashed white outline; numbers = filled circle
   + white ring + `context.draw(Text("\(v)").font(.system(size: d*0.52, weight: .bold)).foregroundColor(.white), at: center)`.
4. `.contentShape(Rectangle())` +
   `DragGesture(minimumDistance: 0).onChanged/.onEnded` → view-model.
5. `.onHover` push/pop `NSCursor.crosshair`.

### Editor/EditorWindowController.swift

`static func open(image: NSImage)`:
- Guard-init the view model (bail silently on nil).
- `NSWindow(contentViewController: NSHostingController(rootView: EditorView(model:)))`,
  title "Snapcat", `titlebarAppearsTransparent = true`. Content size: image
  aspect fitted into 1100×720 plus ~44 pt toolbar height; min 560×400.
- Keep strong refs: static array of controllers; the controller is the window's
  delegate and removes itself in `windowWillClose`.
- `NSApp.activate(ignoringOtherApps: true)` (accessory apps don't auto-activate),
  `showWindow`, `center()`.

## Pitfall checklist (read before coding)

- Carbon callback: C function pointer, no captures — userData dance is required.
- `screencapture` cancellation: no file created → silently return.
- Coordinates: image-pixel top-left EVERYWHERE; only two flips exist in the
  whole app: (a) blur `from:` rect in export, (b) none other — don't invent more.
- `NSImage(size:flipped:true)` drawing handler + `NSAttributedString.draw` is
  the sanctioned combo; no `lockFocus`.
- Preview panel must NOT steal focus: `.nonactivatingPanel` + `orderFrontRegardless()`.
- SwiftUI `Shape` conforms to `View` — the blur mask can be used directly in `.mask(...)`.
- Swift 5 mode: no `Sendable`/actor annotations needed; just dispatch UI work to
  main queue in Process/Timer callbacks.
